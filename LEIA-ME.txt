WORKSHOP: O QUE UM GESTOR DE MARKETPLACE FAZ
Landing + Checkout Externo InfinitePay (Vercel)
================================================

ESTRUTURA
  index.html            → landing (visual idêntico à versão anterior)
  confirmado.html       → página pós-pagamento (/confirmado)
  css/style.css         → estilos
  js/main.js            → countdown, animações, sticky e iniciarCheckout()
  api/create-checkout.js→ cria o checkout na InfinitePay e devolve { url }
  api/payment-check.js  → verificação de pagamento (estrutura p/ uso futuro)
  assets/foto-time.jpg  → foto do time
  vercel.json           → cleanUrls (faz /confirmado funcionar sem .html)
  package.json          → Node 18+

DEPLOY NA VERCEL
  1. Suba esta pasta em um repositório no GitHub
  2. Na Vercel: Add New Project → importe o repositório → Deploy
  3. Pronto. Nenhuma configuração extra é necessária.

FLUXO
  Botão "Garantir vaga" → POST /api/create-checkout →
  redireciona para checkout.infinitepay.io → pagamento →
  InfinitePay redireciona para /confirmado → botão do grupo WhatsApp

CONFIGURAÇÕES
  Link do grupo WhatsApp → confirmado.html, linha:
     const WHATSAPP_GROUP_URL = "COLOCAR_LINK_AQUI";

  Preço (centavos), handle e descrição → api/create-checkout.js
  (ou via variáveis de ambiente na Vercel: PRICE_CENTS,
   INFINITEPAY_HANDLE, PRODUCT_DESCRIPTION, BASE_URL)

  Data do countdown → js/main.js, linha:
     const target = new Date('2026-07-16T18:30:00-03:00').getTime();

OBSERVAÇÃO
  O redirect_url é montado automaticamente a partir do domínio do
  deploy (funciona em preview e produção). Se usar domínio próprio,
  defina BASE_URL nas variáveis de ambiente da Vercel.
